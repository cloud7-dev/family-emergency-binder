# 보안 모델

이 프로젝트의 기본값은 local-first입니다. 계정 생성, 서버 업로드 없이 암호화된 vault 파일로 내보내는 방향을 잡습니다.

현재 릴리즈 후보 주의사항:

- 현재 브라우저 구현은 WebCrypto 기반 PBKDF2와 AES-GCM을 사용합니다.
- 아직 보안 감사를 받은 구현이 아닙니다.
- 암호화, 저장, 내보내기 경로가 검토되기 전까지 실제 민감정보를 입력하지 않는 것이 좋습니다.
- 원문 비밀번호, private key, seed phrase, 전체 식별번호 저장은 기본적으로 권장하지 않습니다.
- 첨부파일은 vault 안에 암호화되지만, 스캔본과 파일은 텍스트 힌트보다 더 많은 민감정보를 담는 경우가 많습니다. 가능하면 작고 가려진 파일을 사용하고 별도 백업을 유지하세요.
- 첨부파일 다운로드와 삭제는 vault가 열린 상태에서만 가능하며, 기존 vault 교체 import는 현재 백업을 먼저 안내합니다.
- 백업 검증은 선택한 vault 파일을 복호화해 확인하지만 현재 열려 있는 vault를 교체하지 않습니다.
- 구조화 필드와 inline edit panel은 사용자 입력을 HTML이 아니라 텍스트로 렌더링해야 합니다.
- 가려진 패킷에는 출력 가능 기록의 구조화 필드만 포함됩니다.

검토 예정 항목:

- KDF 파라미터와 Argon2id 지원.
- 복호화 데이터의 메모리 유지 시간.
- 내보내기 redaction.
- clipboard와 logging 동작.
- XSS hardening.
- 첨부파일 복원과 삭제 hardening.
- 백업 우선 import와 백업 검증 흐름.
- 접근성과 다국어 범위.
